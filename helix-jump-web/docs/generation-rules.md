# 무한 플랫폼 생성 규칙 (Procedural Generation Rules)

## 개요
플레이어가 끊임없이 내려갈 수 있도록 플랫폼을 **무한으로 생성**합니다.
난이도가 불공정하게 어려워지는 것을 방지하기 위해 아래 규칙을 적용합니다.

---

## 기본 파라미터

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 플랫폼 간격 | 1.8 ~ 2.2 | 플랫폼 사이의 Y 거리 (랜덤 변동) |
| 구멍 크기 | 70° ~ 100° | 통과 가능한 구멍의 각도 범위 |
| 버퍼 거리 | 15 | 공 위치 아래로 미리 생성할 거리 |
| 정리 거리 | 10 | 공 위치 위로 이 정도 지나면 플랫폼 삭제 |

---

## 구멍 위치 규칙

### 1. 연속성 규칙 (Gap Continuity)
- 다음 플랫폼의 구멍 중심은 이전 플랫폼 구멍 중심에서 **±120도 이내**에 위치해야 함.
- 이유: 플레이어가 너무 급격하게 회전해야 하는 상황 방지.

```
예시:
이전 구멍: 45° ~ 115° (중심: 80°)
다음 구멍 중심 범위: -40° ~ 200° (= 320° ~ 200°)
```

### 2. 함정 플랫폼 규칙 (Trap Rules)
- 함정 플랫폼의 구멍은 **바로 위 플랫폼 구멍과 90도 이내**에 위치해야 함.
- 이유: 회전 없이도 통과할 수 있는 여지를 줘야 공정함.
- 함정 플랫폼은 연속으로 **2개 이상 배치되지 않음**.
- 함정 출현 확률: **15%** (매 6~7개 플랫폼당 약 1개)

### 3. 움직이는 플랫폼 규칙 (Moving Platform Rules)
- 움직이는 플랫폼 출현 확률: **20%**
- 움직이는 플랫폼 바로 다음에는 함정이 오지 않음.
- 이유: 움직임을 예측하기 어려운 상황에서 함정까지 있으면 불공정.

---

## 난이도 진행 (Difficulty Scaling)

플레이어가 내려갈수록(점수가 높아질수록) 난이도가 점진적으로 증가합니다.

| 점수 구간 | 함정 확률 | 움직이는 플랫폼 확률 | 구멍 최소 크기 |
|----------|----------|-------------------|--------------|
| 0 ~ 100 | 10% | 15% | 90° |
| 100 ~ 300 | 15% | 20% | 80° |
| 300 ~ 500 | 20% | 25% | 70° |
| 500+ | 25% | 30% | 60° |

## 색상 규칙

| 플랫폼 종류 | 색상 |
|------------|------|
| 일반 | `0xffffff` (흰색) |
| 움직이는 | `0xce93d8` (연보라) |
| 함정 | `0xef9a9a` (연빨강) |

## 구현 알고리즘

```typescript
function generateNextPlatform(prevGapCenter: number, score: number): PlatformData {
    // 1. 난이도 기반 파라미터 계산
    const difficulty = getDifficulty(score);
    
    // 2. 구멍 중심 결정 (이전 구멍 ±120도 내)
    const gapCenter = prevGapCenter + randomRange(-120, 120);
    
    // 3. 구멍 크기 결정
    const gapSize = randomRange(difficulty.minGapSize, 100);
    
    // 4. 플랫폼 타입 결정 (일반/움직이는/함정)
    const type = determineType(difficulty, prevWasMoving, prevWasTrap);
    
    // 5. 함정인 경우: 구멍을 이전 구멍과 90도 이내로 조정
    if (type === 'trap') {
        adjustGapForTrap(gapCenter, prevGapCenter);
    }
    
    return { gapStart, gapEnd, isTrap, isMoving, color };
}
```

## 메모리 관리

- 공 위치보다 **10 단위 이상 위**에 있는 플랫폼은 Scene에서 제거.
- 공 위치보다 **15 단위 아래**까지 플랫폼이 미리 생성되어 있어야 함.
- 플랫폼 풀(Pool) 방식은 사용하지 않음 (성능 충분).
